# Color Converter - CMYK ↔ LAB ↔ HSV

## Описание проекта

Веб-приложение/десктопное приложение для конвертации цветов между тремя цветовыми моделями: **CMYK ↔ LAB ↔ HSV**. Приложение позволяет интерактивно изменять цвет, показывая его составляющие во всех трех моделях одновременно.

## Функциональность

- **Три цветовые модели**:
  - CMYK (Cyan, Magenta, Yellow, Key) - в процентах от 0 до 100%
  - LAB (Lightness, A, B) - L: 0-100, a/b: -128 до 127
  - HSV (Hue, Saturation, Value) - H: 0-360°, S/V: 0-100%

- **Три способа задания цвета**:
  1. Прямой ввод значений в текстовые поля
  2. Изменение с помощью ползунков
  3. Выбор цвета из стандартной палитры

- **Автоматический пересчет**:
  - При изменении любой компоненты в любой модели все остальные представления автоматически пересчитываются
  - Визуальное отображение текущего цвета

- **Обработка ошибок**:
  - Предупреждения при выходе значений за допустимые границы
  - Автоматическое обрезание/округление некорректных значений

## Установка и запуск

### Требования
- Python 3.6 или выше
- Библиотеки: tkinter (обычно входит в стандартную поставку Python)

### Установка
1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/color-converter.git
cd color-converter
```

2. Установите необходимые зависимости (если требуются):
```bash
pip install numpy
```

### Запуск
```bash
python3 main.py
```

## Сборка в EXE (опционально)

Для создания исполняемого файла под Windows:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed color_converter.py
```

Исполняемый файл будет создан в папке `dist/`.

## Использование приложения

### Интерфейс
1. **Верхняя панель**:
   - Кнопка "Выбрать из палитры" - открывает стандартный диалог выбора цвета
   - Прямоугольник - показывает текущий выбранный цвет

2. **Секции цветовых моделей**:
   - Каждая модель имеет:
     - Метки компонентов (C, M, Y, K для CMYK и т.д.)
     - Текстовые поля для ввода значений
     - Ползунки для плавного изменения

### Как работать
1. **Изменить цвет через CMYK**:
   - Введите значения в поля C, M, Y, K (0-100%)
   - Или используйте ползунки под полями ввода

2. **Изменить цвет через LAB**:
   - Введите значения L* (0-100), a* (-128 до 127), b* (-128 до 127)
   - Или используйте ползунки

3. **Изменить цвет через HSV**:
   - Введите значения H° (0-360), S% (0-100), V% (0-100)
   - Или используйте ползунки

4. **Выбор из палитры**:
   - Нажмите кнопку "Выбрать из палитры"
   - Выберите цвет в появившемся диалоговом окне
   - Все модели автоматически обновятся

### Особенности
- При изменении любого значения происходит автоматический пересчет всех моделей
- При некорректных значениях (например, при преобразовании LAB в RGB) появляется предупреждение
- Значения автоматически ограничиваются допустимыми диапазонами

## Техническая информация

### Алгоритмы преобразования
- **RGB ↔ CMYK**: Используются стандартные формулы с учетом черного канала (K)
- **RGB ↔ LAB**: Преобразование через промежуточное пространство XYZ с использованием белой точки D65
- **RGB ↔ HSV**: Используется библиотека colorsys Python

### Структура кода
- `ColorConverter` - класс со статическими методами для всех преобразований
- `ColorConverterApp` - основной класс приложения с GUI на tkinter
- Основные методы:
  - `update_all_from_rgb()` - централизованное обновление всех моделей
  - Обработчики событий для полей ввода и ползунков
  - Методы для работы с цветовой палитрой

### Поддерживаемые преобразования
Приложение реализует преобразования для варианта №5:
1. CMYK ↔ RGB ↔ LAB
2. CMYK ↔ RGB ↔ HSV
3. Неявно: RGB ↔ XYZ (для LAB преобразований)

## Возможные проблемы и решения

### Проблема: Цвета выглядят некорректно при преобразовании LAB → RGB
**Решение**: LAB пространство шире, чем sRGB. При выходе за границы RGB происходит обрезание значений с соответствующим предупреждением.

### Проблема: Приложение не запускается с ошибкой "No module named 'tkinter'"
**Решение**: Установите tkinter:
- Ubuntu/Debian: `sudo apt-get install python3-tk`
- Windows: Переустановите Python с включенной опцией "tcl/tk"

### Проблема: Ползунки двигаются рывками при быстром перемещении
**Решение**: Это ограничение tkinter. Для более плавной работы можно увеличить производительность компьютера или оптимизировать код преобразований.
